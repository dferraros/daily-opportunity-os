## 7. BUYBOT SPECIFICATION

### Recommended Bot: Rose Bot (or Combot as alternative)

**Rose Bot** (preferred):
- Free tier covers all required functionality
- Auto-welcome with custom message (see Section 3)
- Anti-spam: link filter, keyword block, duplicate message block
- Auto-mute and ban with configurable thresholds
- Reports in group with audit trail

**Combot** (alternative if Rose Bot is insufficient):
- Better analytics dashboard
- Spam score per user
- Slower setup but more configurable

### Required Bot Configuration

**Auto-welcome:** Enable. Use Message 1 from Section 3. Bot posts in group (not DM) on first join.

**Anti-spam rules:**
- Block: messages containing ".eth", ".sol", ".base" addresses from accounts less than 7 days old
- Block: messages with 3+ external links
- Block: messages containing "100x", "guaranteed", "early access whitelist" from non-admin accounts
- Slow mode: 30-second delay per message for accounts less than 24 hours old
- Max message length: No limit (but enforce quality via culture, not character count)

**Admin commands Daniel must know:**
- `/ban [user]` — permanent remove
- `/mute [user] [duration]` — temporary silence
- `/pin [message]` — pin to top
- `/warn [user]` — one strike logged

**Keyword triggers for auto-flag to Daniel:**
- "launch" + "token" (potential creator inquiry — Daniel should respond)
- "invest" (rule violation — redirect)
- "airdrop" (check if it's an Oikos context or unsolicited promotion)
- "rug" (potential FUD or legitimate concern — Daniel should respond)

### Buybot Note
Standard "buybot" (announcing token purchases) is NOT recommended for t.me/oikoscash during Phase 1. Reasons:
1. OKS volume is ~$2/day — buybot announcements would be embarrassingly infrequent
2. Buybot signals token speculation, which contradicts the credibility-first positioning
3. Add buybot only after OKS volume exceeds $500/day or after first token launches on the protocol
