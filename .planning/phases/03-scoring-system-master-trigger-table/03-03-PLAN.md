**13.4 Journey vs Alert Conflict Resolution (CHAN-04)**

When a user is in an active CleverTap Journey (J1-J6) AND a trigger fires simultaneously:

Conflict resolution matrix (from research CHAN-04):

| Scenario | Resolution | Rationale |
|----------|-----------|-----------|
| User in J1-J6 + Family C | SUPPRESS Family C | Behavioral nudge can wait |
| User in J1-J6 + Family D | SUPPRESS Family D | Lifecycle nudge can wait |
| User in J1-J6 + Family E | SUPPRESS Family E | Cross-sell can wait |
| User in J1-J6 + Family B | SEND Family B | Market data is time-sensitive + MiCA simultaneous send |
| User in J1-J6 + Family A | SEND Family A | User explicitly requested |
| User in J1-J6 + Family F | SEND Family F | Protective alert overrides journey |
| Two triggers same family | Send highest send_score only | Avoid double-nudging |
| P2 lifecycle + P3 market same day | Send both | Different families, different caps |

Implementation: `WHERE active_journey IS NULL` in targeting SQL for Families C, D, E. Families A, B, F bypass this check.

Additional conflict rules:
- When multiple triggers of the SAME family fire in the same evaluation window, only the highest send_score sends
- When triggers of DIFFERENT families fire, they are independent (different caps apply)
- Family B simultaneous send rule overrides ALL other conflict logic (MiCA Art. 89 requirement)
