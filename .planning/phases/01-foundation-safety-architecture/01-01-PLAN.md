**2.4 Cooldown Rules** -- Table:
| Condition | Cooldown | Scope | Mechanism |
- Between any two notifications (same channel): 4 hours minimum -- per user -- CleverTap campaign frequency limit
- After dismissed notification: 24 hours for that trigger family -- per user per family -- BigQuery fatigue tracking + CleverTap user property
- After 3 consecutive dismissals (same family): 7-day family cooldown -- per user per family -- BigQuery computes, syncs to CleverTap via Hightouch
- After 5 consecutive dismissals (any family): Reduce to P0-P1 only for 14 days -- per user -- BigQuery sets notification_cooldown=true, synced to CleverTap as exclusion filter
- After user re-engages (opens/clicks): Reset cooldown immediately -- per user -- BigQuery recalculates on next sync

**2.5 Fatigue Risk Score Formula** -- Document exactly:
```
fatigue_risk = (notifications_sent_7d / 5) * 0.4
             + (notifications_dismissed_7d / max(notifications_sent_7d, 1)) * 0.3
             + (days_since_last_open / 7) * 0.3

  fatigue_risk > 0.9:  CRITICAL -- suppress P2+ (only P0, P1 can send) for 7 days
```
